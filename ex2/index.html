2.1 - Confidential client-server comunications

Utilizando a aplicação cliente-servidor desenvolvida no exercício anterior, vamos modificá-la por forma a garantir a confidencialidade nas comunicações. O cliente vai passar a enviar mensagens cifradas ao servidor, que as decifra.

Get Java Client and Server; server and client scripts.

O cliente foi atualizado para enviar mensagens (byte a byte) cifradas ao servidor, que as decifra.

Para escolher uma das várias cifras e modos:

  1. Correr normalmente o servidor com ./server;
  2. RC4: Correr o cliente com ./client ou ./client rc4;
  3. AES/CBC/NoPadding: Correr o cliente com ./client cbc;
  4. AES/CBC/PKCS5Padding: Correr o cliente com ./client cbc_pdd;
  5. AES/CFB8/NoPadding: Correr o cliente com ./client cfb8;
  6. AES/CFB8/PKCS5Padding: Correr o cliente com ./client cbf8_pdd;
  7. AES/CFB/NoPadding: Correr o cliente com ./client cfb;
